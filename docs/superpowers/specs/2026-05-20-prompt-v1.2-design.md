# Prompt v1.2 改进设计

## 目标
改善 prompt 系统，功能操作对齐数据库，强化括号表情禁令，不改动其他模块。

## 改动文件清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `server/prompts/prompt_composer.py` | 修改 | memory 独立预算 + 懒加载 |
| `server/prompts/global_prompt_v1.2.py` | 新建 | 禁令强化、memory_snippets 拆出 |
| `server/prompts/daily_greeting_v1.2.py` | 新建 | 括号禁令、few-shot、注释统一 |
| `server/prompts/memory_chat_v1.2.py` | 新建 | 记忆引用标记、few-shot |
| `server/prompts/cognitive_test_v1.2.py` | 新建 | 空图片处理、few-shot |
| `server/prompts/emotion_support_v1.2.py` | 新建 | positive_topics 标注数据源 |
| `lib/logic/prompt_task_router.dart` | 修改 | emotion 分支补传 positive_topics |
| `lib/logic/chat_provider.dart` | 修改 | memory_snippets 提到 prompt_context 顶层 |
| `server/prompts/test_composer_v12.py` | 新建 | v1.2 冒烟测试 |

## 一、prompt_composer.py 改造

### 1.1 memory 独立预算
- 当前 `budget_memory: 300` 定义但未使用
- 改为三块拼接：global(≤1200) + task(≤900) + memory(≤300) → 总计兜底 ≤2400
- `compose_for_request` 优先读 `prompt_context["memory_snippets"]`（新），回退 `prompt_context["global"]["memory_snippets"]`（旧）

### 1.2 懒加载
- 当前 import 时一次性加载全部 10 个 .py
- 改为存 (filename, func_name) 元组 + 运行时 `_MODULE_CACHE` 按需加载
- 加载函数沿用现有 `importlib.util.spec_from_file_location` 方式

## 二、5 个 Prompt 模块改进

### 2.1 global_prompt v1.2
- 禁令第 7 条（括号表情）→ 提升到第 1 条，加正确/错误对照示例
- 去掉 `memory_snippets` 入参和 `_clip_list`
- `elder_profile_brief` 注释标注数据源：users.hometown/career/hobbies/personality/birth_year/name

### 2.2 daily_greeting v1.2
- 末尾加括号禁令 + few-shot 话术示范
- 数据库操作注释统一为 `-- Flutter 端操作: LocalDatabase.xxx(...)`
- 涉及表：daily_life_records (breakfast/lunch/dinner/activities/people_met/places_went/mood)

### 2.3 memory_chat v1.2
- few-shot 对照（自然关联 vs AI 式查档案）
- 末尾加括号禁令
- 涉及表：memory_events 读 (listMemoryEventsForUser) + 写 (touchMemoryEventUsage)

### 2.4 cognitive_test v1.2
- image_path 为空时：用 `PROMPT_TEMPLATE_IDLE` 告知模型本轮不出题
- few-shot + 括号禁令
- 涉及表：cognitive_tests (countCognitiveTestsToday/getLastCognitiveTestTime/getRecentInvalidStreak/insertCognitiveTest)

### 2.5 emotion_support v1.2
- positive_topics 参数注释标注数据源
- 查询：`SELECT title,description FROM memory_events WHERE owner_user_id=? AND (emotion IN('开心','喜悦','满足') OR importance>=4) ORDER BY importance DESC LIMIT 4`
- 括号禁令

## 三、Flutter 端改动

### 3.1 chat_provider.dart
- `memory_snippets` 从 `global` 内移到 `prompt_context` 顶层

### 3.2 prompt_task_router.dart
- `_buildEmotionParams` 改为 async，新增 `_queryPositiveMemories` 查 memory_events
- 传给 emotion_support prompt 的 `positive_topics` 不再为空

## 四、数据库对齐验证

| 模块 | 操作 | 表/列 | 验证 |
|------|------|-------|------|
