# Prompt v1.3 改进设计

## 目标
解决模型违反括号禁令的问题，提升对话流畅性和调用成功率。核心发现：prompt 中的负例（"差"示范）包含括号模式，反向教会了模型使用括号。

## 改动原则
- **模型输出什么，取决于它看到了什么** — 洗掉 prompt 里所有括号模式，只用纯正例
- 括号禁令首尾双保险（首因+近因效应）
- `---` 分隔符改为自然过渡
- 服务端正则过滤作为最后安全网
- 新建 v1.3 文件，保留 v1.0/v1.1/v1.2 不动

## 一、5 个 Prompt 模块 — 去掉所有负例

### 1.1 global_prompt v1.3
- 删除"绝对禁令"第 1 条中的错误示例（含 `（微笑）`、`（关心地）`）
- 改为：禁令语句 + 纯正例（不展示任何括号写法）
- 其他禁令和结构保持不变

改写前：
```
1. 不用括号表达动作、表情或心理活动；情绪和语气直接融入口语里。
   错误：「您今天气色不错（微笑）」
   正确：「您今天气色不错，我看着就高兴。」
   错误：「那您多注意身体（关心地）」
   正确：「您可得注意身体，别累着。」
```
改写后：
```
1. 不用括号表达动作、表情或心理活动。情绪和语气直接融入口语里。
   比如想说"您今天气色不错"时，直接说"您今天气色不错，我看着就高兴"，
   而不是加括号标注。温暖和关心都写在话里，不需要额外标记。
```

## 二、prompt_composer.py 改造

### 2.1 分隔符自然化
- `CONFIG["separator"]` 从 `"\n\n---\n\n"` 改为 `"\n\n"`
- 双换行足够区分 block，`---` 割裂整体感

### 2.2 尾部格式提醒（近因效应）
- 在 `compose_system_prompt` 的 `merged` 末尾统一追加：
  ```
  【输出格式】不用括号表达动作或表情。
  ```
- 各 task block 末尾的"风格提醒"行删除，避免重复

### 2.3 注册 v1.3
- `_VERSION_LOADERS` 新增 `"v1.3"` 条目，指向 5 个 v1.3 模块
- 默认 version 改为 `"v1.3"`

## 三、服务端正则安全网

`local_chat_server.py` 新增函数，在模型回复返回给 Flutter 前自动过滤：

```python
import re

_BRACKET_PATTERN = re.compile(r'（[^）]*）')

def _strip_brackets(text: str) -> str:
    return _BRACKET_PATTERN.sub('', text).strip()
```

在 `/api/chat` 响应处理中，对 assistant 的 `content` 字段调用此函数。即使 prompt 失效，括号也会被物理删除。

## 五、预期效果

- **流畅性**：去掉 `---` + 纯正例 → prompt 像一个人说的话而不是检查清单
- **可靠性**：首尾双保险（禁令在首 + 格式提醒在尾）+ 正则兜底 → 两层防护
- **可回退**：旧版本全部保留，一行代码即可切回 v1.2
